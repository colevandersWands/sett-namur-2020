un pas d'avance

7 conseils


(début)

1.

familiarizer vous avec les malcompréhentions fréquants

many misunderstandings don't result in bad assignments
there are some very common ones you can come prepared for
this will also help you learn enormously

2.

choisissez d'abord l'environnement,
puis le language

changing is very disruptive
many of the questions you'll get are about where they code, not even the code
this is an oportunity to relevantize to your domain/subject

3.

pratiquez le debugging
plus que le coding

students can be left to figure out their own solutions
but being stuck on errors is not so motivating or productive
it will also give you much more confidence, in my experience
it's easier to say with a straight face when i can't actually help:
you're here to learn how to solve these tough problems
than, you're here to spend 20 minutes staring at broken errors

4.

commencez avec du code fini

small chagnes are big changes
repeated similar tasks (as long as they aren't boaring) are effecive
better-contain questions & problems to things you're familiar with
writing code from scratch is far from the only good way to learn it
and may even be a bad way
either code they, you or someone else has written
draw diagrams to explain the code
give a simple starter program and have pairs go back-and-forth adding to it
write stories about where/how it's helpful
...

5.

concentrez-vous sur le process,
pas le produit

6.

pseudo-data au lieu du pseudo-code

practice stepping through the solution process on multiple examples
instead of talking about possible solution code
more immediate, better look 'behind the scenes'
more transferable outside of coding

7.

utilise les platforms déjà existant

you already know how difficult it is to scope a lesson plan and stay within it
this is at least as hard with programming
don't hesitate to fall-back on existing curriculums & materials
they'll be a self-contained ecosystem
and can leave you more headspace to focus on integrating with your subject
